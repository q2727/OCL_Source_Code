# OCL 论文复现实验报告

论文: *Categorical Data Clustering via Value Order Estimated Distance Metric Learning*
arXiv: `2411.15189v5`

---

## 1. 复现范围

### 方法

| 方法 | CLI 名 | 范式 | 说明 |
|------|--------|------|------|
| OCL | `ocl` | 划分式 | 完整 OCL（概率加权 order distance + 迭代 order learning） |
| KModes | `kmd` | 划分式 | 传统 k-modes，Hamming 距离 |
| KMedoids | `kmedoids` | 划分式 | PAM 算法，overlap 距离 |
| Agglomerative | `agg` | 层次式 | 重叠距离 + average linkage |
| Spectral | `spectral` | 图方法 | 重叠相似度 + 归一化 Laplacian |
| OCL-I | `ocl1` | 消融 | 等距 order distance，无概率加权 |
| OCL-II | `ocl2` | 消融 | OCL-I + 仅一次 order 更新 |
| OCL-III | `ocl3` | 消融 | Hamming 距离，无 order 信息（等价 KMD） |

### 指标

| 指标 | 类型 | 范围 | 说明 |
|------|------|------|------|
| CA | 外部 | [0,1] ↑ | Hungarian 最优匹配准确率 |
| ARI | 外部 | [-1,1] ↑ | 调整兰德指数 |
| NMI | 外部 | [0,1] ↑ | 归一化互信息 |
| CMP | **内部** | [0,1] **↓** | 聚类紧密度（Eq.17），归一化熵均值 |

### 数据集

- 主数据集 12 个: `AC/AP/BC/CS/DS/HR/LG/NS/SB/TT/VT/ZO`
- 补充数据集 6 个: `AV/BM/CC/CR/CT4/OB`
- 缺失 2 个: `HF`、`MA/Mat`（仓库无 `.mat` 文件）

```bash
.venv/Scripts/python -m ocl_python.cli --all --runs 10 --seed 25 --methods ocl,ocl1,ocl2,ocl3

```

种子序列: `seed=25..34`（10 次重复）。论文未公开精确种子序列，做趋势对照。

## 6. 消融实验

### 变体定义

| 变体 | Order Distance | 概率加权 | 迭代更新 Order | 等价于 |
|------|---------------|----------|---------------|--------|
| OCL | ✅ 概率加权 | ✅ | ✅ | 完整算法 |
| OCL-I | ✅ 等距 | ❌ 硬 mode | ✅ | 去掉概率加权 |
| OCL-II | ✅ 等距 | ❌ 硬 mode | ❌ 仅一次 | OCL-I 基础上去迭代 |
| OCL-III | ❌ Hamming | ❌ | ❌ | = KMD |

### 12 数据集平均

| 变体 | CA | Δ vs OCL | ARI | NMI |
|------|-----|----------|------|------|
| OCL | 0.6512 | — | 0.2730 | 0.2760 |
| OCL-I | 0.6174 | -0.0338 | 0.2346 | 0.2436 |
| OCL-II | 0.6074 | -0.0100 | 0.2202 | 0.2367 |
| OCL-III | 0.5875 | -0.0199 | 0.1970 | 0.2196 |

**每个组件的贡献**:
- 概率加权距离: **ΔCA = -0.034**（最大贡献）
- 迭代更新 order: **ΔCA = -0.010**
- Order 信息本身: **ΔCA = -0.020**

### 单数据集注意

**HR 异常**: OCL (CA=0.362) < OCL-II (0.411) < OCL-III (0.385)。在 HR 上，迭代概率加权 order learning 反而误导了聚类——这是算法局限性的直接证据。

**收敛曲线可视化**: 见 `results/figures/convergence_*.png`，展示了目标函数下降和 order update 节点的位置。

## 7. 参数敏感性

对 `max_outer_loops` ∈ {1,2,3,5,10,20,50} 和 `max_init_loops` ∈ {1,2,5,10,20,50} 做网格搜索。

关键发现:
- **SB/NS**: 1–2 次 outer loop 即达到最优，对参数不敏感
- **VT**: 性能高度稳定，所有参数组合 CA 均在 0.876–0.894 之间
- **ZO**: outer≥5、init≥5 后结果趋于稳定
- **结论**: 默认的 `max_outer_loops=50`、`max_init_loops=50` 非常保守，实际只需 5–10 次即可收敛

## 9. 新 Baseline 的原创发现

1. **Agglomerative Clustering 在某些数据集上显著超越 OCL**:
   - SB: CA=1.000 (完美), ZO: CA=0.881, DS: CA=0.825
   - 但这些数据集样本量小（SB=47, DS=120），在大数据集（NS=12960）上 Agg 表现平庸
   - 说明对于小样本分类数据，层次聚类可能是更好的选择

2. **Spectral Clustering 在 LG 上 NMI 最高**（0.22），展示了图方法在特定数据结构上的优势

3. **OCL 并非普适最优**: 在 HR (CA=0.362) 和 NS (CA=0.329) 上，OCL 甚至不如 KMD

## 10. 偏差原因排查

1. **随机种子不一致** — MATLAB `rand('seed', 25)` ≠ NumPy `RandomState(25)`。DS 数据集已验证: seed=1000 时 CA=0.825（接近论文 0.746），而 seed=25 时 CA=0.674

2. **DS 对初始化极敏感** — 5 个二值属性 + k=2 的粗粒度距离空间使 OCL 易陷入"不动点"

3. **数据集缺失** — HF 和 MA/Mat 无 `.mat` 文件，20 数据集只跑了 18 个

4. **浮点精度差异** — MATLAB 和 NumPy 的浮点计算在 tie-breaking 时可能产生不同结果

5. **CMP 已实现** — 修正了初版误实现为聚类纯度的错误，现使用论文 Eq.17 的熵归一化公式
