# 当前支持实验结果与论文趋势对照

本文档记录当前 Python 版可直接运行的实验范围、实际运行结果，以及和论文 v5 结果的趋势对照。

论文来源：`https://arxiv.org/html/2411.15189v5`

## 1. 本次复现范围

当前仓库可运行的方法：

- `KMD`：传统 k-modes baseline，使用 Hamming 距离。
- `OCL`：从 MATLAB 迁移的 order-based categorical clustering 主算法。

当前仓库可运行的数据：

- `OCL_Source_Code/Data`：12 个主数据集，分别是 `AC/AP/BC/CS/DS/HR/LG/NS/SB/TT/VT/ZO`。
- `OCL_Source_Code/Supp_data`：6 个补充数据集，分别是 `AV/BM/CC/CR/CT4/OB`。
- 论文 v5 表中还有 `HF` 和 `MA/Mat`，当前仓库没有对应 `.mat` 文件，所以本次没有运行。

当前可对照指标：

- 已实现：`CA`、`ARI`、`NMI`。
- 未实现：论文 v5 里的 `CMP`。

## 2. 实验命令

主数据集：

```bash
.venv/bin/python -m ocl_python.cli --all --runs 10 --seed 25 --methods all
```

补充数据集：

```bash
.venv/bin/python -m ocl_python.cli --all --data-root OCL_Source_Code/Supp_data --runs 10 --seed 25 --methods all
```

等价的 `uv` 命令是：

```bash
uv run ocl-run --all --runs 10 --seed 25 --methods all
uv run ocl-run --all --data-root OCL_Source_Code/Supp_data --runs 10 --seed 25 --methods all
```

本次运行使用 `seed=25..34` 作为 10 次重复实验的随机种子序列。论文只说明取 10 次平均，没有公开精确种子序列，因此这里做趋势对照，不要求数值完全相等。

## 5. 总体趋势与收益

18 个可运行数据集的平均值：

| Method | CA | ARI | NMI |
| --- | --- | --- | --- |
| KMD | 0.5648 | 0.1415 | 0.1608 |
| OCL | 0.6027 | 0.1943 | 0.2012 |
| OCL - KMD | +0.0379 | +0.0528 | +0.0404 |

OCL 的收益主要体现在 ARI/NMI，而不只是 CA：

- `KMD` 用 Hamming 距离，所有不同类别值的距离都相同，无法表达同一属性内不同取值之间的相近或相斥关系。
- `OCL` 会根据当前聚类分布学习每个属性值的 order，再用 order distance 聚类。直观上，它让更能代表同一簇的取值在编码空间里更靠近，让簇间区分强的取值更远。
- 因此 OCL 往往能得到更一致的簇结构。这个收益在 `SB`、`AC`、`AP`、`BC`、`VT`、`ZO` 等数据集上很明显。
- 补充数据集里 CA 不一定总是提升，但 ARI/NMI 多数提升，说明 OCL 有时没有提高最佳标签匹配准确率，却改善了整体成对关系和信息一致性。

## 6. 和论文 v5 的一致性

论文 v5 的实验设置和本次一致的部分：

- 聚类数 `k` 使用真实类别数。
- 每个结果使用 10 次运行平均值。
- 当前对照使用论文表中的 `CA`、`ARI`、`NMI`，暂不包含 `CMP`。

当前 OCL 与论文 OCL 在 18 个可运行数据集上的平均差值：

| Metric | Current OCL - Paper OCL |
| --- | --- |
| CA | -0.0241 |
| ARI | -0.0248 |
| NMI | -0.0165 |

按绝对误差 0.03 粗略判断：

- CA：10/18 个数据集在 0.03 以内。
- ARI：11/18 个数据集在 0.03 以内。
- NMI：14/18 个数据集在 0.03 以内。

这说明当前结果整体趋势与论文接近，尤其 NMI 最稳定；但 CA/ARI 在部分数据集上仍有明显偏差。

偏差较明显的数据集：

- CA 明显低于论文：`AC`、`DS`、`HR`、`SB`、`AV`、`CC`、`CR`、`CT4`。
- ARI 明显低于论文：`AC`、`DS`、`HR`、`LG`、`NS`、`SB`、`AV`。
- NMI 明显低于论文：`AC`、`DS`、`HR`、`NS`。

## 7. 如果不符合论文效果，优先排查什么

1. 随机种子不完全一致。
   MATLAB 使用 `rand('seed', 25)` 和 `randperm`，Python 使用 NumPy `RandomState`。即使 seed 数字相同，初始 mode 序列也不会逐项一致。论文也没有公开 10 次运行的具体种子序列。

2. baseline 实现范围不同。
   当前 `KMD` 是仓库内实现的传统 k-modes/Hamming baseline，用来做趋势对照。它不保证等同于论文作者使用的 KMD 实现、重启策略或 tie-breaking。

3. 数据集版本与预处理可能不同。
   论文 v5 使用 20 个数据集，当前仓库只有 18 个对应 `.mat` 文件。即使缩写对应，v5 论文中的预处理版本也可能和仓库 MAT 文件不完全一致。

4. 指标范围尚不完整。
   当前实现了 CA/ARI/NMI，未实现 CMP，所以不能完全复刻论文 v5 的全部表格。

5. OCL 迁移细节要继续做 MATLAB 逐步对齐。
   本次已修正一个关键差异：MATLAB 第一次 INOCL 前的 `ModeMtx` 是 mode point 的 one-hot 分布，而不是初始化 partition 的经验分布。若要进一步缩小偏差，下一步应在 MATLAB 可运行环境里逐数据集比较初始化 modes、每轮 partition、order、objective history。
